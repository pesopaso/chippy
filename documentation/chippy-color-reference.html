Chippy — Colour reference v1.0

Every colour the app exposes, grouped by role. The surface scale stays neutral so the accents read cleanly. The page is split into three blocks: a dark-mode block at the top, a shared block in the middle for colours that look the same in both themes, and a light-mode block at the bottom — each clearly labelled so it's never ambiguous which mode a swatch belongs to.

Background

Outermost canvas behind everything. Sits behind the sidebar and the main column; not surfaced through any chrome.
Surface
#20293a--surface

  • Sidebar fill
  • Modal dialog backgrounds

  • Right-panel section containers

#334253--card
Inset elements inside a surface.
  • Task rows in the right panel
  • Tag chips

  • Secondary buttons
  • Avatar circles in the member list
Border
#46556b--border
Divider lines and edges, plus a deliberately-quiet semantic role.
  • Panel borders and section dividers
  • Chrome-bar bottom rule
  • Input field outlines
  • "Never used" tag colour in overviews
Text
#e3e7f0--text
Primary body text. Comments, task titles, history entries, anything the reader is meant to focus on.
Muted
#92a3b9--muted
Secondary text and metadata. Reads as text but recedes.
  • Timestamps
  • Entry counts
  • Filter input placeholders
  • Second-line context inside cards

Pink
#b32794--pink
One of the discussion-tag rotation colours. The sidebar cycles through this hue plus accent / green / yellow / red / purple / orange to colour-code discussion groups when more than one tag is in use.
Followup
#d9a3bc--followup
Followup entries — left-border stripe on followup rows, and chip backgrounds. A decoupled variant so it can be tuned independently of the discussion-name colour.
Goal
#225a7a--goal
Goal entries — left-border stripe on goal cards in the right panel and the All Goals list. Distinct from accent so goal cards stay visually separate from task cards.

Top chrome — hardcoded, identical in both modes

Top chrome
#0c18a0not a CSS variable — inline hex
The bar across the top of the viewport. Holds:
  • App title + version
  • Recent-discussion tabs
  • Print button (context-aware)
  • Theme toggle
  • Help icon
Hardcoded rather than themed because it stays the same in both light and dark mode.

In-progress.

Needs human review.
  • The skill writes this when it fails or aborts
  • User clicks it manually to flag "look at me"
  • A lightened mauve for better contrast at small sizes

#e19a25
Parked. Not blocked, just not active. New tasks added via the skill's capture mode default here for the human to gate.

Purgatory. Halfway between obsolete and on-hold — kept visible but not actively prioritized.

#317b61

Theme-aware Accents that differ between dark and light

Theme-dependent colours — same CSS variable, different value per theme

Dark Light
Accent
D#518eaa L#384057 --accent
The app's primary action colour. Light mode uses a darker, less saturated blue so it reads cleanly against the white surface.
  • Active nav buttons
  • Save button fill
  • Link colour
  • Focus rings on inputs
  • Hover targets and selected states
Dark Light
Name
D#93b4ac L#214937 --name
In-text @[Name] references render in this colour. Inline coloured text without a chip box so reading flow isn't interrupted. Light mode uses a much darker green so the references remain legible against the paper-white surface.
